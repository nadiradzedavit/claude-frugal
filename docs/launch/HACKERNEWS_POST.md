I built this after repeatedly hitting long-session friction in Claude Code.

The current package installs three skills:

- slim-read: a small Python helper that lists file structure or extracts one symbol/range
- distill: a prompt workflow for producing a compact handoff before /compact
- limit-watch: rough context-size guidance

The project is intentionally labeled beta. The repo includes a theoretical token
calculator, but I am not treating the modeled numbers as proof. I am looking for
real reports from people using Claude Code on normal projects.

Feedback I would value most: whether the skill format works for you, whether
slim-read actually changes your workflow, and whether the docs are clear.
